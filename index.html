The structure of an HTML document

  • Every HTML document that conforms to the HTML specification of any version, must start with the HTML version declaration line <!DOCTYPE ...> which usually looks something like this: <!DOCTYPE html>. In this case we are talking browser that HTML conforms to the international specification version 5.

  • No need to try to remember this line. The main thing is to know that it is necessary. After all, it helps to make a site that will look the same in all browsers (usually Mozilla Firefox, Google Chrome, Opera, Internet Explorer).

  • Next, the beginning and end of the document are marked with <html> and </html> tags, respectively. Inside these tags should be the head (<head></head>) and document body (<body></body>) tags. 

    <!DOCTYPE HTML> 
<html> 
<head> 
<title>The structure of an HTML document</title>
</head>
<body> 
<h1>Hello HTML!</h1> 
</body> 
</html>
Header of the HTML page

The title of the page is the text that is displayed in the upper left corner of the browser, as well as in the name of the tabs. To set the page title, you need to use the <title> tag inside the <head> tag. In the previous example, 

<head>
<title>The structure of an HTML document</title>
</head>
HTML page coding

  • The encoding of an HTML page must be specified so that the web browser can correctly display the text on the page. If the browser incorrectly "guesses" the encoding, characters will be displayed instead of text. To inform the browser of the coding of the HTML page, it is necessary to use the tag inside the <head> tag: 

    <meta charset="encoding name">

  • The most common modern encoding is utf-8. Use it in all your projects. For Cyrillic in Windows, the charset was often set as windows-1251. But now it is considered a bad practice.

Key words

  • There is a whole family of <meta> service tags, which are called "meta tags". They are used inside the <head> tag. Meta tags are distinguished by a set of attributes and their values. Here are some of the attributes: "content", "http-equiv", "name", and "scheme".

  • Meta tags store useful information for browsers and search engines. One such tag is a description of the page's keywords. It is set as follows: 

    <meta name="keywords" content="different, key, words">

  • In the content attribute, the most important words from the content of the page are listed through a comma. Earlier, this tag was very important for search engines. What is the state of affairs now - the big secret of search engines.

Description of page content

  • Another useful meta tag for search engines is a short description of the page. It is set as follows: 

    <meta name="description" content="brief description">

  • The content attribute should contain a summary or abstract of the page, which is often used by search engines when displaying search results.

HTML comments

By the way, you can see the following code fragments in the HTML editor: 

<!-- Some text -->

These are called "comments" and the browser does not display them on the page.

Comments are usually used in the following cases:

  • for commenting the code (it is always useful to leave a hint);

  • to temporarily disable the code (deleting the code is inconvenient, as it will have to be restored, and commenting and then uncommenting is the best solution).

Comments can be used anywhere on the page except the <title> tag - inside which they do not work.

Basic elements of text markup

The main elements of text formatting in html are tags:

  • p - is used to divide the text into paragraphs;

  • h1, h2, ..... h6 - are used to create headings of 1,2 ... 6 levels, respectively;

  • br - used to move a line;

  • div, span - використовуються для виділення частини документа певним способом.

The given tags are far from a complete list of elements with which the test web page is formatted. But they may be enough to create it.
